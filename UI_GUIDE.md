# UI GUIDE

## 1. Visual Direction
Theme name: **Calm Workshop**

Meaning:
- Calm colors
- Clean layout
- Friendly text
- Easy reading for beginners

## 2. Original Design Rules
- Do not copy any existing tools website design
- Use custom wording and custom section structure
- Keep cards and spacing consistent
- Focus on clarity over decoration

## 3. Color System (CSS Variables Plan)
- `--bg-main`: light warm background
- `--bg-card`: card background
- `--text-main`: primary text
- `--text-soft`: secondary text
- `--brand-main`: primary action color
- `--brand-accent`: accent color
- `--border-soft`: soft border color
- `--success`: success feedback color
- `--danger`: error feedback color

## 4. Typography Plan
- Heading font: modern, rounded sans-serif
- Body font: clean, highly readable sans-serif
- Keep large line-height for easier reading
- Use short paragraphs for beginner users

## 5. Layout and Spacing Rules
- Use container max width for readability
- Keep clear white space between sections
- Use 8px spacing system (8, 16, 24, 32, 40, 48)
- Keep cards with same border radius and padding

## 6. Reusable UI Components (Planned)
- `SiteHeader`
- `SiteFooter`
- `SectionTitle`
- `ToolCard`
- `CategoryCard`
- `SearchBar`
- `FilterChips`
- `ToolShell` (common layout wrapper for tool pages)
- `ToolInputPanel`
- `ToolOutputPanel`
- `FAQBlock`
- `RelatedTools`

## 7. Tool Page UX Rules
- Main action button should be very clear
- Copy button should be visible near output
- Reset/Clear button should be easy to find
- Show friendly validation errors (simple words)
- Show small privacy note for browser-only processing

## 8. Mobile Responsive Rules
- Mobile-first layout
- Single-column cards on small screens
- Touch-friendly button sizes
- Sticky or easy-to-reach actions when useful
- No text should overflow on small screens

## 9. Accessibility Basics
- Use semantic HTML tags
- Keep color contrast readable
- Add clear labels for inputs
- Add keyboard-focus styles
- Add alt text for meaningful images/icons

## 10. AdSense-Ready Layout Note
- Reserve clean optional blocks in future
- Do not add ads now
- Keep content-first structure
